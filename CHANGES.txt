================================================================================
                            HOW THE FIX WORKS
================================================================================

DELETE OPERATION:
-----------------
Old Flow:
  User deletes file → Mark deleted → Update quota by -filesize → Done
  Problem: Delta could be wrong if file record had incorrect size

New Flow:
  User deletes file → Mark deleted → Recalculate from database → Done
  Benefit: Always accurate because calculated from actual files in DB


STORAGE DISPLAY:
----------------
Old Flow:
  Get storage → Check fm_user_storage_tracking → Empty table → Show 0 B

New Flow:
  Get storage → Check fm_user_storage_tracking → If empty, check fm_user_quotas
               → Return accurate data from whichever table has it
  Benefit: Always shows correct values


GLOBAL STATISTICS:
------------------
Old Issue:
  Global VPS usage summed fm_user_storage_tracking which was empty → 0 B

New Solution:
  Sum from fm_user_storage_tracking, fallback to fm_user_quotas
  Both tables kept in sync by fm_recalculate_user_storage()
  Benefit: Global stats always accurate
